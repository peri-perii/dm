03.1

Huffman Coding

Optimal Prefix-Free Data Compression Algorithm

📌 Definition

Huffman Coding is a lossless data compression algorithm that assigns variable-length binary codes to characters based on their frequencies. Characters with higher frequencies receive shorter codes, and those with lower frequencies receive longer codes. It produces optimal prefix-free codes.

🔑 Key Properties

✓
Prefix-Free

No codeword is a prefix of another. Enables unambiguous decoding.

✓
Greedy Optimal

Produces minimum expected code length — provably optimal.

✓
Variable Length

Different characters get different length codes based on frequency.

✓
Lossless

Original data can be perfectly reconstructed from compressed form.

⚙️ Algorithm Steps

01
Create Leaf Nodes

Create a leaf node for each character with its frequency.

02
Build Priority Queue (Min-Heap)

Insert all leaf nodes into a min-heap ordered by frequency.

03
Merge Two Minimum Nodes

Extract the two nodes with minimum frequency. Create a new internal node with frequency = sum of both. Make them left and right children.

04
Insert & Repeat

Insert the new internal node back into the heap. Repeat step 3 until only one node remains — the root.

05
Assign Codes

Traverse the tree: assign 0 for left branch, 1 for right branch. Code for each character = path from root to leaf.

📐 Worked Example — From PYQ

Characters with frequencies: a(4), b(3), c(2), d(7), e(14), f(9), g(13), h(11), k(5)

1Sort: c(2), b(3), a(4), k(5), d(7), f(9), h(11), g(13), e(14)
2Merge c(2)+b(3) = N₁(5). Heap: a(4), k(5), N₁(5), d(7), f(9), h(11), g(13), e(14)
3Merge a(4)+k(5) = N₂(9). Heap: N₁(5), d(7), f(9), N₂(9), h(11), g(13), e(14)
4Merge N₁(5)+d(7) = N₃(12). Heap: f(9), N₂(9), h(11), N₃(12), g(13), e(14)
5Merge f(9)+N₂(9) = N₄(18). Continue until single root...
Average Code Length = Σ(frequency × code_length) / Σ(frequency)
Total characters = 4+3+2+7+14+9+13+11+5 = 68

📊 Complexity

Time ComplexityO(n log n)
Space ComplexityO(n)
Best CaseAll frequencies equal → Complete binary tree
Compression RatioTypically 20-90% depending on data

🧠 Key Formulas

Average Bits per Character
Lavg = Σ fᵢ × lᵢ / Σ fᵢ
fᵢ = frequency, lᵢ = code length
Compression Ratio
CR = Lfixed / Lhuffman
L_fixed = ⌈log₂ n⌉ bits for n symbols
03.2

Kruskal's Algorithm

Minimum Spanning Tree via Greedy Edge Selection

📌 Definition

Kruskal's Algorithm finds a Minimum Spanning Tree (MST) of a connected, undirected, weighted graph. An MST is a spanning subgraph (includes all vertices) that is a tree and whose total edge weight is minimized.

Spanning Tree: A tree that includes all vertices of the graph.
MST: Spanning tree with minimum total edge weight.

⚙️ Algorithm

01
Initialize

Create a forest where each vertex is a separate tree. T = ∅ (empty MST).

02
Sort All Edges

Sort all edges E in non-decreasing order of weight.

03
Process Edges Greedily

For each edge (u,v) in sorted order: if u and v are in different components (no cycle), add edge to T and merge their components using Union-Find.

04
Terminate

Stop when T has exactly n-1 edges (where n = number of vertices). T is the MST.

Algorithm Kruskal(G=(V,E,w)):
  Sort E by weight: e₁ ≤ e₂ ≤ ... ≤ eₘ
  Initialize Union-Find for each v ∈ V
  T ← ∅
  
  for each edge eᵢ = (u,v) in sorted order:
    if Find(u) ≠ Find(v):     // different components
      T ← T ∪ {eᵢ}
      Union(u, v)
    if |T| = |V| - 1: break
  
  return T

🔑 Union-Find Data Structure

📁
Make-Set

Create singleton set {v} for each vertex v

🔍
Find

Return root of component containing v

🔗
Union

Merge two components into one

⚡
Path Compression

Optimization: flatten tree during Find

📐 Worked Example

Graph: Vertices {A,B,C,D,E}, Edges: A-B(5), A-C(6), B-D(3), C-D(5), C-E(8), D-E(8)

StepEdgeWeightActionMST Edges So Far

2A-B5Add (no cycle){B-D, A-B}
3C-D5Add (no cycle){B-D, A-B, C-D}
4A-C6Skip (creates A-B-D-C-A cycle)—
5C-E8Add (no cycle) — 4th edge, DONE{B-D, A-B, C-D, C-E}
MST Total Weight = 3+5+5+8 = 21

📊 Complexity & Properties

Time ComplexityO(E log E) or O(E log V)
Space ComplexityO(V + E)

03.3

Handshaking Theorem

Fundamental Degree-Sum Formula in Graph Theory

📐 The Theorem

Handshaking Theorem
Σv∈V deg(v) = 2|E|
The sum of degrees of all vertices equals twice the number of edges.

🔍 Explanation

Each edge (u, v) contributes exactly 1 to deg(u) and 1 to deg(v), so it contributes 2 to the total degree sum. Hence, summing over all vertices counts every edge exactly twice.

Intuition: At a party of n people, if each handshake involves exactly 2 people, then the total number of handshakes received = twice the number of handshakes performed.

🔑 Corollaries

C1
Even number of odd-degree vertices:
In any graph, the number of vertices with odd degree is always even.
Proof: Σ deg(v) = 2|E| (even). Even sum minus sum of even-degree terms = sum of odd-degree terms = even → must have even count of odd-degree vertices.
C2
For regular graphs (k-regular):
If every vertex has degree k: n·k = 2|E| → |E| = nk/2
(This requires nk to be even, so either n or k must be even)
C3
Directed graphs:
Σ in-deg(v) = Σ out-deg(v) = |E|
Each directed edge contributes 1 to in-degree of head and 1 to out-degree of tail.

📐 Applications

🔗
Eulerian Circuits

Eulerian circuit exists iff every vertex has even degree (from Handshaking, this means |E| = Σdeg(v)/2 must make all deg(v) even)

📊
Edge Counting

Count edges without explicitly listing them: |E| = Σdeg(v)/2

✅
Proof Verification

Verify a claimed graph is consistent: degree sequence must sum to even number

🎯
Network Analysis

Used in circuit analysis, network flow, and social network degree distribution

📝 Exam-Style Problems

Example 1: A graph has 6 vertices with degrees 1,1,2,2,3,3. Does it satisfy Handshaking?
Σdeg = 1+1+2+2+3+3 = 12 = 2×6 ✓ → 6 edges. Valid graph.
Example 2: Can a graph have 5 vertices each with degree 3?
Σdeg = 5×3 = 15 (odd) ≠ 2|E|. Impossible! No such graph exists.
Example 3: Tutorial 7, Q1: Σdeg(a,b,c,d) = 3+3+3+3 = 12 = 2×6 ✓

Trees — Complete Guide

Every Tree Concept, Property, and Algorithm

📌 What is a Tree?

A tree is a connected, undirected graph with no cycles (acyclic). Equivalently, a tree on n vertices has exactly n-1 edges.

Tree Equivalences
A graph T is a tree iff any two of the following hold (the third follows):
1) T is connected  |  2) T is acyclic  |  3) T has n-1 edges

🌳 Types of Trees

Free Tree

An unrooted tree. No designated root. Purely structural — edges are undirected.

n vertices n-1 edges Connected Acyclic
Rooted Tree

One vertex designated as the root. Gives a parent-child hierarchy. All edges directed away from root.

Root vertex Parent/Child Level/Height
Binary Tree

Each vertex has at most 2 children (left and right). Used in BSTs, heaps, expression trees.

≤2 children/node Left subtree Right subtree
Full Binary Tree

Every vertex is either a leaf or has exactly 2 children. No vertex has only one child.

0 or 2 children 2L-1 = internal nodes
Complete Binary Tree

All levels completely filled except possibly the last. Last level filled left to right.

Height = ⌊log₂ n⌋ Used in Heaps
Perfect Binary Tree

All internal nodes have exactly 2 children AND all leaves are at the same level.

2^(h+1)-1 nodes 2^h leaves
m-ary Tree

Each vertex has at most m children. Binary tree is 2-ary. Used in B-trees, file systems.

≤m children Full m-ary: 0 or m
Spanning Tree

A spanning subgraph of a graph G that is a tree. Contains all vertices of G with n-1 edges.

All vertices Subgraph of G No cycles
Minimum Spanning Tree

Spanning tree with minimum total edge weight. Found by Kruskal's or Prim's algorithm.

Min total weight Kruskal / Prim
Binary Search Tree (BST)

Binary tree where left subtree has smaller values, right subtree has larger values than root.

Left < Root < Right Search: O(log n)
Expression Tree

Binary tree representing arithmetic expressions. Internal nodes = operators, leaves = operands.

Preorder=Prefix Inorder=Infix Postorder=Postfix
Decision Tree

Binary tree used for decision making. Each internal node = condition, leaf = decision/outcome.

Yes/No decisions Classification

  1. Visit Node (root)
  2. Traverse Left subtree
  3. Traverse Right subtree
Used for: Copying trees, Prefix notation
Inorder (LNR)
  1. Traverse Left subtree
  2. Visit Node (root)
  3. Traverse Right subtree
Used for: BST sorted output, Infix notation
Postorder (LRN)
  1. Traverse Left subtree
  2. Traverse Right subtree
  3. Visit Node (root)
Used for: Deleting trees, Postfix (RPN) notation
Level Order (BFS)
  1. Visit root
  2. Visit all level-1 nodes (left to right)
  3. Visit all level-2 nodes, etc.
Used for: Shortest path, level-by-level processing

🔍 Binary Search Tree Operations

🔍
Search(T, k)

If T is empty → not found. If k = T.key → found. If k < T.key → Search(T.left, k). If k > T.key → Search(T.right, k). Time: O(h) where h = height.

➕
Insert(T, k)

Find correct position via search, insert as new leaf. If k < root → go left, if k > root → go right. Time: O(h).

➖
Delete(T, k)

Case 1: Leaf → simply remove. Case 2: One child → replace with child. Case 3: Two children → replace with inorder successor (leftmost in right subtree).

🕸️ Spanning Trees

A spanning tree of a connected graph G = (V, E) is a tree T = (V, E') where E' ⊆ E, |E'| = |V| - 1, and T connects all vertices.

🌊
BFS Spanning Tree

Built using Breadth-First Search. Gives shortest paths (unweighted) from source.

🏊
DFS Spanning Tree

Built using Depth-First Search. Reveals back edges, used for cycle detection.

⚖️
Kruskal's MST

Globally greedy edge selection. O(E log E). Best for sparse graphs.

📍
Prim's MST

Locally greedy vertex expansion. O(E log V). Best for dense graphs.

Number of Spanning Trees of Kₙ (Complete Graph): n^(n-2)  (Cayley's Formula)
Examples: K₃ → 3, K₄ → 16, K₅ → 125
03.5

Finite State Machines

DFA, NFA, Moore & Mealy Machines

📌 Definition

A Finite State Machine (FSM) / Finite Automaton (FA) is a mathematical model of computation used to design digital sequential logic and recognize patterns in strings.

Formal Definition — DFA
M = (Q, Σ, δ, q₀, F)
Q = finite set of states  |  Σ = input alphabet
δ: Q×Σ→Q = transition function  |  q₀ ∈ Q = initial state
F ⊆ Q = set of accepting (final) states

🔄 Types of Finite Automata

DFA — Deterministic Finite Automaton

For each state and input symbol, there is exactly one next state. δ: Q×Σ→Q is a total function.

Deterministic No ε-transitions Unique next state
NFA — Nondeterministic Finite Automaton

For each state and input, there may be zero, one, or multiple next states. Also allows ε-transitions.

Nondeterministic ε-transitions allowed Power set states
Moore Machine

Output depends only on the current state. Output function λ: Q→Γ. Used in synchronous sequential circuits.

Output on state M = (Q,Σ,Γ,δ,λ,q₀)
Mealy Machine

Output depends on the current state AND input. Output function λ: Q×Σ→Γ. More general than Moore.

Output on transition M = (Q,Σ,Γ,δ,λ,q₀)

📐 FSM Representation

🔵
State Diagram

Directed graph. Vertices = states. Labeled edges = transitions. Double circle = accept state. Arrow = start state.

📋
Transition Table

Rows = states, Columns = input symbols, Entries = next state(s). Compact representation.

📝
Transition Function

δ: Q×Σ→Q listed explicitly as ordered pairs. δ*(q, w) = extended function for strings.

📊
State Table

Same as transition table but may include output column for Moore/Mealy machines.

🔑 Acceptance & Language

A string w ∈ Σ* is accepted by DFA M if δ*(q₀, w) ∈ F (ends in an accept state).

The language recognized: L(M) = {w ∈ Σ* | δ*(q₀, w) ∈ F}

Equivalence: DFA and NFA recognize the same class of languages — Regular Languages. Every NFA can be converted to an equivalent DFA (Subset/Powerset construction).

🔄 NFA to DFA Conversion (Subset Construction)

01
Compute ε-closure

ε-closure(q) = all states reachable from q using only ε-transitions (including q itself).

02
Initial DFA State

Start state = ε-closure({q₀}). Each DFA state is a set of NFA states.

03
Compute Transitions

For each DFA state S and symbol a: δ_DFA(S, a) = ε-closure(∪_{q∈S} δ_NFA(q, a))

04
Accept States

A DFA state S is accepting if S ∩ F_NFA ≠ ∅ (contains at least one NFA accept state).

Concatenation uv: Append v directly after u with no separator.

|uv| = |u| + |v| = 5 + 4 = 9 characters

Concatenation is not commutative: uv ≠ vu in general

vu = abba · aabaa = abbaaabaa ≠ uv

T5·Q2
Languages

Let L₁ = {a, b}, L₂ = {a², ab, b²}, L₃ = {b, b², b³, …}. Find L₁L₂, L₁L₃, L₂L₃.

Solution — Language Concatenation
L₁L₂ — all strings formed by one string from L₁ followed by one from L₂:
L₁L₂ = {a·a², a·ab, a·b², b·a², b·ab, b·b²} = {aaa, aab, ab², baa, bab, bb²} = {aaa, aab, abb, baa, bab, bbb}
L₁L₃ — prefix one symbol from {a,b} before any b-power string:
L₁L₃ = {ab, ab², ab³, …, bb, bb², bb³, …} = {a·bⁿ | n≥1} ∪ {b·bⁿ | n≥1} = {abⁿ | n≥1} ∪ {bⁿ | n≥2}
L₂L₃ — concatenate any string from L₂ with any string from L₃:
L₂L₃ = {a²bⁿ | n≥1} ∪ {abbⁿ | n≥1} ∪ {b²bⁿ | n≥1} = {a²bⁿ | n≥1} ∪ {abⁿ | n≥2} ∪ {bⁿ | n≥3}
Key: Language concatenation L₁L₂ = {uv | u ∈ L₁, v ∈ L₂}. It is generally not commutative.
